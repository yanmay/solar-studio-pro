Create NASA POWER Edge Function   [NEW FILE]
 
Create a new file at api/solar-data.ts in the project root (NOT inside src/).
This is a Vercel Edge Function that proxies NASA POWER API calls server-side.
Requirements:
- Accept GET requests with query params: lat (number), lng (number), year (optional number, default to current year minus 1)
- Validate that lat and lng are present and are valid numbers. Return 400 with JSON error if missing.
- Construct a NASA POWER API request to:
  {process.env.NASA_POWER_BASE_URL}/api/temporal/climatology/point
  with params: parameters=ALLSKY_SFC_SW_DWN, community=RE, longitude={lng}, latitude={lat}, format=JSON
- Fetch this URL server-side using the native fetch API.
- On success, extract the monthly GHI array from the response at:
  properties.parameter.ALLSKY_SFC_SW_DWN
  and return it as JSON: { monthly_ghi: [...12 values...], lat, lng, year }
- On NASA API error, return 502 with { error: "NASA API unavailable", details: <status> }
- Set response header: Cache-Control: public, max-age=86400 (cache for 24 hours)
- Export the handler as default export compatible with Vercel Edge runtime.
- Use TypeScript with explicit types. No any types.
- Do not import anything from src/. This file is standalone.
  ↳  Check the file lands at api/solar-data.ts, not src/api/solar-data.ts.
  ↳  The Cache-Control header is important — it stops Vercel from re-fetching NASA on every map interaction.

Create Google Geocoding Edge Function   [NEW FILE]
 
Create a new file at api/geocode.ts in the project root (NOT inside src/).
This is a Vercel Edge Function that proxies Google Geocoding API calls server-side.
Requirements:
- Accept GET requests with query param: q (string, the address to geocode)
- Validate that q is present and non-empty. Return 400 with JSON error if missing.
- Construct a Google Geocoding API request to:
  https://maps.googleapis.com/maps/api/geocode/json
  with params: address={encodeURIComponent(q)}, key={process.env.GOOGLE_GEOCODING_KEY}
- Fetch this URL server-side.
- On success, extract the first result and return:
  { lat, lng, formatted_address, place_id }
  sourced from results[0].geometry.location and results[0].formatted_address
- If Google returns status !== "OK" or results is empty, return 404 with { error: "Address not found" }
- On fetch failure, return 502 with { error: "Geocoding service unavailable" }
- Set response header: Cache-Control: public, max-age=3600
- Export as default Vercel Edge Function.
- TypeScript with explicit types. No any types.
- Do not import anything from src/.
  ↳  Geocoding results are deterministic for the same address — 1hr cache is fine and saves quota.
  ↳  The key difference from the current setup: GOOGLE_GEOCODING_KEY (no VITE_ prefix) stays server-side only.

Create payment/verify Edge Function   [NEW FILE]
 
Create a new file at api/payment/verify.ts in the project root.
This Edge Function verifies the Razorpay payment signature after the user pays.
Requirements:
- Accept POST requests only. Return 405 for other methods.
- Parse the JSON body as VerifyPaymentRequest (see src/types/payment.ts for shape):
  { razorpay_order_id, razorpay_payment_id, razorpay_signature, plan, scanId? }
- Verify the HMAC SHA-256 signature using the Node.js crypto module:
  const body = razorpay_order_id + '|' + razorpay_payment_id
  const expectedSignature = createHmac('sha256', process.env.RAZORPAY_KEY_SECRET)
                              .update(body).digest('hex')
  const isValid = expectedSignature === razorpay_signature
- If isValid is false, return 400 with { error: 'Invalid payment signature' }
- If isValid is true, return 200 with { verified: true, paymentId: razorpay_payment_id }
- Import createHmac from the 'crypto' module (Node built-in, no install needed)
- TypeScript, no any types, default export.
- Do not import anything from src/.
CRITICAL: The body string for HMAC must be exactly: order_id + '|' + payment_id
with a pipe character separator. This is Razorpay's required format.
  ↳  This is the most security-critical function in the app. If the HMAC check is wrong, anyone can fake a payment.
  ↳  After Antigravity writes this, manually read the HMAC logic and verify the pipe separator is correct.
