SCN-003
Edge Function: Payment Verify (HMAC Signature Check)
Backend / Security
MUST-HAVE
DESCRIPTION
Create a Vercel Edge Function at api/payment/verify.ts that verifies the Razorpay HMAC signature after a payment completes, then sets the secure unlock cookie. This is the most security-critical function in the app — a wrong implementation allows anyone to unlock reports without paying.
CONTEXT
Razorpay generates a cryptographic signature for every payment using your webhook secret. Your server re-computes the same signature and compares. If they match, the payment is real. If not, someone is trying to fake a payment.
DEPENDS ON
→  SCN-002
NOTES / WARNINGS
⚠  CRITICAL: The pipe separator | in the HMAC body string is Razorpay's required format. Any deviation causes all verifications to fail.
⚠  Test this with Razorpay's test mode before touching production keys.
ACCEPTANCE CRITERIA  —  this ticket is done when all of the following are true
1.  File exists at api/payment/verify.ts in the project root.
2.  POST with valid razorpay_order_id, razorpay_payment_id, razorpay_signature returns HTTP 200 { verified: true, paymentId }.
3.  POST with an invalid/tampered signature returns HTTP 400 { error: 'Invalid payment signature' }. No cookie is set.
4.  On success, response sets a cookie: scan_unlocked_<scanId>=true; HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=86400.
5.  The HMAC body string is exactly: razorpay_order_id + '|' + razorpay_payment_id (pipe separator, no spaces).
6.  RAZORPAY_KEY_SECRET is never included in any response.
AI CODING PROMPT  —  copy and paste this directly into Antigravity
Create a new file at api/payment/verify.ts in the project root.

This Edge Function verifies a Razorpay payment signature and sets an unlock cookie.

Requirements:
- Accept POST requests only. Return 405 for other methods.
- Parse JSON body: { razorpay_order_id: string, razorpay_payment_id: string, razorpay_signature: string, plan: string, scanId?: string }
- Verify the HMAC signature using Node's built-in crypto module:
    import { createHmac } from 'crypto'
    const body = razorpay_order_id + '|' + razorpay_payment_id
    const expected = createHmac('sha256', process.env.RAZORPAY_KEY_SECRET!)
                       .update(body).digest('hex')
    const isValid = expected === razorpay_signature
- If isValid is false: return 400 { error: 'Invalid payment signature' } — do NOT set any cookie.
- If isValid is true:
    1. Create a Response with body { verified: true, paymentId: razorpay_payment_id }
    2. Set header on the response:
       Set-Cookie: scan_unlocked_<scanId>=true; HttpOnly; Secure; SameSite=Strict; Path=/; Max-Age=86400
       where <scanId> is the scanId from the request body (use 'default' if not provided)
    3. Return the response with HTTP 200
- On any unexpected error: return 500 { error: 'Verification failed' }
- TypeScript, no 'any' types. Import createHmac from 'crypto' (Node built-in, no install needed).
- Do NOT import from src/.

CRITICAL: The HMAC body string MUST be exactly order_id + pipe + payment_id with no spaces.
Getting this wrong means all legitimate payments will be rejected.
